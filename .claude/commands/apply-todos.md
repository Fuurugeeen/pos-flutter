# apply-todos コマンド

doc/todo.md の TODO 項目に基づいて、実際のコード修正を実施します。

## タスク手順

1. **doc/todo.md の内容を確認**
   - 未着手または進行中の TODO 項目を抽出
   - 優先度と依存関係を確認
   - 実施可能な項目を特定

2. **各 TODO 項目の実施**
   - 対象ファイルを特定して読み取る
   - 必要な修正内容を理解する
   - コードの変更を実施
   - 変更内容を検証

3. **修正内容のチェック**
   - コードが正しく動作するか確認
   - エラーや警告がないか確認（flutter analyze）
   - 必要に応じてテストを実行（flutter test）
   - リンティングエラーがないか確認

4. **コミットと TODO 更新**
   - 変更内容をコミット（問題がない場合）
   - doc/todo.md の該当項目を完了済みに更新
   - 完了日時と実施内容を記録

5. **再確認と次のステップ**
   - 全ての TODO 項目が対応済みか確認
   - 未対応の項目があれば理由を報告
   - 新たに発見された課題があれば TODO に追加

## 注意事項

- 一度に複数の TODO を処理する場合、依存関係を考慮する
- エラーが発生した場合は作業を中断し、ユーザーに報告
- 大きな変更は小さな単位に分割してコミット
- テストが失敗した場合は、TODO を完了済みにしない
- 作業ログを doc/ フォルダに出力する（CLAUDE.md のルールに従う）

## コミットメッセージフォーマット

以下の形式でコミットメッセージを作成：

```
<type>: <変更内容の要約>

- 詳細な変更内容1
- 詳細な変更内容2

🤖 Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>
```

type: feat, fix, refactor, test, docs, style など
