# process-todos コマンド

todo-tmp.md から todo.md への統合と、TODO 項目の実施を一括で実行します。

このコマンドは以下の2つのコマンドを順次実行します：

## 実行フロー

### ステップ 1: /sync-todos の実行
1. doc/todo-tmp.md の修正指示を精査
2. doc/todo.md に反映
3. todo-tmp.md の項目を完了済みにマーク
4. 検証と確認

### ステップ 2: /apply-todos の実行
1. doc/todo.md の TODO 項目を確認
2. 各項目の修正を実施
3. 修正内容をチェック
4. コミットと TODO 更新
5. 全体の確認

## 実行前の確認事項

- doc/todo-tmp.md に修正指示が記載されているか確認
- バックアップが必要な場合は事前に取得
- 未コミットの重要な変更がないか確認

## 実行中の処理

1. **同期フェーズ**（sync-todos）
   - todo-tmp.md の全項目を読み取る
   - todo.md に統合する
   - 完了マークを付ける

2. **実施フェーズ**（apply-todos）
   - 未着手の TODO を順次処理
   - 各変更をテストとコミット
   - TODO を完了済みに更新

3. **最終確認**
   - todo-tmp.md の全項目が完了済みか確認
   - todo.md の全項目が対応済みか確認
   - 未対応の項目があれば理由を報告

## エラーハンドリング

- エラーが発生した場合は、その時点で処理を中断
- エラー内容とスタックトレースを報告
- 部分的に完了した作業の状態を明示
- 再実行可能な状態を維持

## 完了条件

以下の全てが満たされた場合に完了とする：
- ✅ todo-tmp.md の全項目が完了マーク済み
- ✅ todo.md の全項目が対応済みまたは保留理由が明記
- ✅ 全てのコード変更がコミット済み
- ✅ テストとリンティングがパス

## 注意事項

- 大量の TODO がある場合、処理に時間がかかる可能性がある
- 重要な変更は必ず事前にコミットまたはバックアップを取る
- 途中で中断する必要がある場合は、Ctrl+C で中断可能
